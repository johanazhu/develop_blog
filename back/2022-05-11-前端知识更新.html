前端知识更新

前端技术更新很快，这里持续记录一些技术点

Monorepo

  • 单一代码库，意味着把所有项目的代码统一维护再一个单一的代码版本库中
  • 与之对应的是 multirepos（多代码库），每个项目都存储再一个完全独立、版本控制的代码中
  • 优点
    • 可见性
    • 唯一依赖源
    • 一致性
    • 共享时间线
    • 原子提交
    • 更简单的依赖关系管理
    • 统一C1/CD
    • 统一的构建流程
  • 缺点
    • 性能差
    • 破坏主线
    • 学习曲线
    • 所有权
    • 大量的数据
    • Code reviews
  • 那些代码库在使用
    • Babel
    • React
    • Umi

pnpm

中文官网：https://www.pnpm.cn/

英文官网：https://pnpm.io/

速度快、节省磁盘空间的软件包管理器

擅长依赖管理

  • 目前最好的包管理工具，pnpm 吸收了 npm、yarn、lerna 等主流工具的精华，并去其糟粕

  • 生态、社区活跃且强大

  • 可结合 workspace 可以完成 monorepo 最佳设计和实践

一个老牌的 node.js 包管理器，支持 npm 的所有功能，用来替代 npm。它的模块是全局存储，每个项目内部使用硬链接，所以很省空间，而且安装速度快

turbo

官网：https://turborepo.org/

它是一个高性能构建系统，拥有增量构建、云缓存、并行执行、运行时零开销、任务管道、精简子集等特性

具有非常有效的任务编排能力，可以弥补 pnpm 在任务编排上的短板
